# Distributed Log Monitoring & Debugging System

## 📌 Overview
This project is a distributed log monitoring system designed to collect, parse, store, and analyze logs from multiple services. It simulates a microservices environment where logs are generated by different services and centralized for monitoring and debugging.

---

## 🚀 Features
- Log ingestion via REST API
- Log parsing and validation
- Structured log storage in MySQL
- Error detection and alert generation
- Log analytics and system health metrics
- Simulation of multiple services generating logs

---

## 🏗️ Architecture

Logger Clients → Flask API (app.py) → Parser → Database → Analytics & Error Detection

---

## 🧰 Tech Stack
- Python
- Flask (REST API)
- MySQL
- Regular Expressions (Log Parsing)
- Requests (Client simulation)

---

## 📁 Project Structure
- app.py → Main API server
- parser.py → Log parsing logic
- database.py → Database connection and queries
- error_detector.py → Error detection and alerting
- analytics.py → Log analysis and metrics
- logger_client.py → Simulates distributed services
- config.py → Configuration settings
- requirements.txt → Dependencies
